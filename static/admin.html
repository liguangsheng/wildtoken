路由、自动权重与重试

这些参数由服务端保存，并用于保存后的新请求。

全局
  1. 模型优先先保留模型匹配分最高的渠道；显式指定渠道时跳过池内权重选择。
  2. 优先级硬分层只要最高 priority 组内存在有效权重大于 0 的渠道，就不会使用更低 priority。
  3. 同层按权重随机自动权重开启时，路由票数为基础权重 × 健康分；关闭时按基础权重固定分流。
  4. 降到 0 后切换某一层全部为 0 才回退到下一层。渠道恢复为正数后，高优先级层会立即重新接管流量。

健康分范围为 0–100。所有上游非 2xx、连接失败、超时、读取失败和 SSE 上游异常都扣分；正常完成的 2xx 请求加分。健康分为 0 时退出自动池，经过完整恢复周期后增加恢复分值并重新参与选择。

自动路由的每次重试都会重新选择渠道。选到不同渠道立即重试；再次选到同一渠道才等待。显式指定渠道的重试始终等待。一旦下游响应头已经提交，尤其是成功的 SSE 流，后续异常只记失败，不再透明重试。
